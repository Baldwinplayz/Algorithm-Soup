This sorting algorithm depends on the javascript "sort()" function. Depending on your browser and javascript engine, the speeds and algorthims might vary. Beware that it will treat all non numerical string as NaN (Not a number), so there will be no sorting in that case. In general the fastest possible sorting algorithm in a specific language like javascript will be the build-in one. Often times this will be because of the increased amounts of optimizations that these function recieve over costum functions; as a result, if speed is a priority one should pick this page.
Due to the fact that this page is generic sorting there will be a list and brief descriptions of some other sorting algorithms:

  1. Quick sort: quick sort is often regarded as the fasted algorthim for sorting, it works under the divide and conquer paradime.
  2. Merge sort: merge sorts works by the divide and conquer paradime, it will sort parts individually and then it will merge them.
  3. Naive sort: this algorthims takes the greatest or smallest number and puts it at the start repeatedly, its very slow as it works on quadratic time.
  4. Bubble sort: this sorting algorthims is one of the slowest, it consists of comparing 2 indexes at a time, it works on quadratic time.
