Description

This algorithm employs the most efficient searching algorithm known: binary search. This algorithm will take a sorted array and use that to its advantage to rule out entire halves of said array; for example, if one is looking for a 5 in the array, and it sees that the number in the center is greater than 5, one will know that if the 5 is present, it must be in the first half. Due to its divide and conquer paradime it runs on O(log n) time. To make this work, you need to sort the array in UNICODE order that can be found using the search bar.
